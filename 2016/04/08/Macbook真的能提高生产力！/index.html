Macbook真的能提高生产力！

发表于 2016-04-08   |   分类于 编程之路   |     |   本文总阅读量 次

在知乎呆了很久，一直听说macbook的种种优势，直到本周三换上一台mbp840，经过两天的体验，只能感叹mac诚不欺我。在这两天的初试中，mac体现出了三大优势：续航，unix环境，良好的软件生态。

续航：

支持我换mac最大的动力之一就是续航，之前使用的联想Y50在ubuntu系统下续航时间也不到两个小时，这就导致我很少带电脑出去，并且在11点熄灯后不能支撑偶尔熬夜的需求。换了mac第二天我便带着电脑出去上选修，从中午到晚上回来，还剩50%多的电，一下午写了两篇markdown文档，看了一下午的pdf，续航能力果真强悍。

unix环境：

OS x良好的继承了unix的环境，作为一个大部分时间在写python，node的我来说，unix比windows好上一百倍。况且os x10.11的扁平化风格正符合我的胃口，比起原来用的ubuntu整体UI更加美观。

软件生态：

在其他的linux或者unix环境下，你很少能看到软件厂商专门为其开发的软件。而这点在mac下得到良好的补充。虽然比不上windows，但各种软件绝对已经满足开发人员的需求。

 

除了以上几点，mac还有很多优点，比如Retina屏，比起1080P的确实看的舒服。其他的我就不提了，最重要的是提高生产力的工具。

提高生产力的工具

说道提高生产力，好的工具是必备的，虽然我们不能依赖工具，但使用好的工具无疑能减少我们大量的工作和时间的浪费。

  • Reeder RSS软件，将自己关注的blog和新闻全部添加进去
  • MindNode Pro 思维导图，比起mind更喜欢MNP的简洁
  • iterm2+oh-my-zsh 谁用谁知道
  • sublime 用了快两年，真的很好用。配置各种插件很关键
  • pycharm 写python最后的ide
  • 马克飞象 + 印象笔记 用了很多markdown软件，最后还是选择马克飞象，与印象笔记的结合让笔记更加灵活。
  • Alfred
  • xcode
  • preview mac自带的pdf阅读器
  • brew 相当于apt-get
     

工具是死的，人是活的。利用好工具，让工作学习更加方便快速。

MacBook Pro真的赞！
